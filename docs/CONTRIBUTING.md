# Contribuindo

## Padroes de documentacao

- Mantenha `SKILL.md` conciso; mova detalhes para `references/`.
- Evite duplicar conteudo entre arquivos.
- Use kebab-case em novos docs de referencia (ex: `references/rate-limits.md`).

## Mudancas e revisoes

- Descreva a intencao, arquivos alterados e novas referencias nos PRs.
- Use mensagens de commit curtas no imperativo (ex: "Add API endpoints doc").

## Mudancas de escopo

Se voce adicionar codigo ou scripts:
- Crie a pasta `scripts/` e atualize `SKILL.md` e `docs/DEVELOPMENT.md`.
- Documente novas variaveis de ambiente.
